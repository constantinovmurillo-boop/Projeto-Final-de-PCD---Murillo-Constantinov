## Descrição do projeto
<p align="justify"> Projeto final desenvolvido para a disciplina de Práticas em Ciências de Dados no primeiro semestre de 2026. A proposta visa, através da linguagem de programação Python, o cálculo de diversas áreas matemáticas. Dividido em 13 módulos, que podem ser importados conforme a necessidade do usuário; sendo esses de constantes, aritmética, trigonometria, funções, números complexos, álgebra, álgebra linear, cálculo, combinatória e probabilidade e estatística.


## Instruções para instalação
<p align="justify"> Para utilizar o projeto, clone o repositório em sua máquina, garantindo a existência de um ambiente compatível com a linguagem Python, como o JupyterLab. Em seguida, execute as células de cada notebook para que os códigos sejam transformados em scripts na sua máquina. Com isso, é possível abrir uma nova célula e importar as funções ou os módulos que necessitar para fazer os cálculos. Atentar-se à ordem de execução dos notebooks*

## Avisos importantes antes da utilização do projeto
- Os valores muitas vezes são aproximações, pois o cálculo computacional é limitado, portanto são usadas série e limites de interações para gerar valores.
  
- Caso exista a necessidade de aproximações mais precisas, pode-se configurar no código de constantes.ipynb.

- *Ao importar os módulos, eles possuem dependências uns dos outros, por isso a ordem de execução de cada notebook deve ser:
    1. Constantes
    2. Aritmética
    3. Trigonometria
    4. Funções
    5. Números complexos
    6. Álgebra
    7. Álgebra linear
    8. Cálculo
    9. Combinatória e probabilidade
    10. Estatística
  
- Ao importar os módulos, nem todas as funções são úteis para o processo, importe apenas as que irá usar (opcional).
  
- Para realizar as operações deve-se sempre usar o formato "algoritmo(número)".
  
- Ao importar objetos definidos pelas classes, sempre use o formato "Objeto(valores)", atentando-se sempre a suas estruturas únicas.

- A geração de números pseudoaleatórios não é criptográfica, portanto não é segura para tais usos.

- As funções possuem restições matemáticas, portanto, caso haja erro ao executar uma operação, leia o erro pois há uma indicação de qual é o problema.

- Funções mais complexas possuem maior complexidade, portanto seu custo computacional é maior

- Essa é uma primeira versão do projeto, como consequência, podem haver erros ou bugs não identificados.

- Como primeira versão, o projeto não faz cálculo simbólico.
